7. 문제 해결

오류 문구를 그대로 확인한 뒤 설치, 로그인, MCP 연결, 실행 세션 중 어느 단계에서 멈췄는지 찾습니다.

Ouroboros 0.50.4upstream 456a134Ubuntu 24.04 · Claude Code 2.1.2102026-07-16 확인

7.1 ooo 명령을 찾을 수 없음

ooo를 일반 터미널에 입력했는지 먼저 확인합니다. 이 가이드의 ooo interview 같은 명령은 로그인한 Claude Code 세션 안에서 사용합니다.

claude plugin list

ouroboros@ouroboros가 enabled가 아니면 다시 설치합니다.

claude plugin install ouroboros@ouroboros --force

Not logged in · Please run /login이 나오면 Claude Code에서 /login을 먼저 마칩니다. 이 가이드의 실측도 이 지점까지 진행했습니다.

7.2 Interview 후 run 또는 status 실패

plugin 기능은 읽혔지만 Core의 MCP 서버가 연결되지 않은 상태일 수 있습니다. Claude Code에서 /mcp로 Ouroboros 상태를 확인하고 순서대로 진행합니다.

ooo setup
ooo update

setup 뒤 Claude Code를 완전히 닫고 다시 엽니다. standalone 설치라면 터미널에서 아래 정보를 확인합니다.

ouroboros mcp info
ouroboros status health

7.3 권한 오류로 파일 수정 실패

runtime의 로그인과 프로젝트 폴더 쓰기 권한을 확인합니다. Ouroboros는 runtime의 승인 장치를 우회하지 않습니다. 읽기 전용 폴더라면 쓸 수 있는 작업 폴더로 옮기고 Seed의 경로도 갱신합니다.

7.4 업데이트 후 명령 변경

ooo update를 실행하고 세션을 다시 엽니다. 이 글 상단의 확인 버전과 현재 버전이 다르면 최신 upstream의 Getting Started와 CLI Reference를 기준으로 삼습니다.

7.5 중단된 auto 재개

ooo auto가 A등급 게이트나 중간 오류로 멈추면 출력에 auto_session_id가 남습니다. 같은 자동 세션을 이어가려면 재개 명령을 실행합니다.

ooo auto --resume <auto_session_id>

ID를 잃었다면 아래의 실행 세션 복구 절차로 중단된 세션을 먼저 찾습니다.

7.6 닫힌 창의 실행 복구

ooo resume-session

EventStore를 직접 읽어 running이나 paused 세션을 보여줍니다. MCP 서버가 꺼져 있어도 읽을 수 있습니다. 원래 Seed 파일이 있다면 표시된 session ID로 재개합니다.

ouroboros run workflow --orchestrator --resume <session_id> <seed.yaml>

상태만 먼저 보고 싶다면 ouroboros tui monitor를 엽니다.

7.7 실행 취소

ooo cancel

활성 실행 목록에서 고르거나 execution ID를 전달합니다. 셸에서는 ouroboros cancel execution <execution_id>을 사용합니다. 취소 뒤 ooo status로 상태를 확인합니다.

7.8 같은 실패 반복

Stagnation은 반복 실행을 해도 결과가 나아지지 않는 상태입니다. 같은 구조와 피드백이 반복되면 실행을 멈추고 접근 방법을 바꿉니다.

ooo unstuck

현재 문제와 이미 시도한 방법을 함께 전달합니다. unstuck은 구현, 조사, 단순화, 구조, 반대 검토의 다섯 관점에서 다른 해결 방법을 제안합니다. 사용할 방법은 사람이 선택합니다.

7.9 제거

먼저 삭제 범위를 미리 확인합니다.

ouroboros uninstall --dry-run

ouroboros uninstall은 MCP 등록, 프로젝트 설정과 ~/.ouroboros/ 데이터를 제거합니다. 데이터를 남기려면 --keep-data를 사용합니다. Claude 플러그인은 별도로 제거합니다.

claude plugin uninstall ouroboros
