8. 용어사전

가이드에 나오는 제품 용어를 기능별로 설명합니다. 각 설명 끝의 장 링크에서 실제 사용 절차를 확인할 수 있습니다.

Ouroboros 0.50.4upstream 456a1342026-07-16 확인

8.1 명세와 Seed

8.1.1 Interview

아이디어에 숨어 있는 가정과 빠진 결정을 질문으로 찾는 과정입니다. 3장

8.1.2 Seed

목표, 제약, 완료 기준, 데이터 구조를 담은 실행 기준 파일입니다. 코딩 도구는 이 파일을 기준으로 작업합니다. 4장

8.1.3 Goal

이번 실행에서 만들어야 할 결과를 한 문장으로 고정한 것입니다. 4장

8.1.4 Constraint

실행이 반드시 지켜야 할 제한입니다. 기술, 환경, 범위가 들어갑니다. 4장

8.1.5 Non-goal

이번에는 만들지 않기로 합의한 일입니다. 4장

8.1.6 Acceptance Criterion

작업이 끝났는지 확인하는 조건입니다. 테스트하거나 화면에서 직접 확인할 수 있는 문장으로 작성합니다. 5장

8.1.7 Ambiguity Score

목표, 제약, 성공 기준에 남은 모호함입니다. 0에 가까울수록 분명하며, 자동 실행을 허용하는 관문은 0.2 이하입니다. 3장

8.1.8 A등급

ooo auto가 실행을 시작하기 전에 Seed가 받아야 하는 등급입니다. 통과하지 못하면 Seed를 수정하거나 실행을 중단합니다. 3장

8.1.9 Ontology

프로젝트가 다루는 대상과 관계를 정한 구조입니다. 할 일 앱에서는 할 일의 제목과 완료 상태가 여기에 포함됩니다. 4장

8.1.10 Ledger

질문과 명세 작성 과정에서 확인한 가정, 제외 범위, 결정 근거를 저장한 기록입니다. 4장

8.2 실행과 평가

8.2.1 Core

Interview, Seed, 실행 조율, 평가와 기록을 담당하는 Ouroboros 엔진입니다. 2장

8.2.2 runtime

Core의 요청을 받아 실제 파일과 명령을 다루는 AI 코딩 도구입니다. Claude Code, Codex CLI 등이 있습니다. 2장

8.2.3 MCP

runtime이 Ouroboros Core의 도구를 호출하게 잇는 통신 규약입니다. 2장

8.2.4 Double Diamond

저장소 확인, 문제 정의, 구현 설계, 코드 작성 순서로 진행하는 실행 구조입니다. 5장

8.2.5 Drift

현재 결과가 원래 Seed의 Goal, Constraints, ontology에서 얼마나 멀어졌는지 나타냅니다. 5장

8.2.6 Mechanical / Semantic / Consensus

빌드와 테스트, 목표와 완료 기준 충족 여부, 추가 판단이 필요한 항목을 차례로 확인하는 평가 단계입니다. 5장

8.3 수정과 반복

8.3.1 Wonder와 Reflect

평가 뒤 아직 모르는 것을 찾고, 다음 Seed에서 무엇을 바꿀지 제안하는 과정입니다. 6장

8.3.2 lineage

같은 시작 Seed에서 이어진 여러 번의 수정 실행을 묶은 기록입니다. 6장

8.3.3 Convergence

반복 실행을 해도 데이터 구조가 거의 바뀌지 않는 상태입니다. ontology 유사도 0.95가 기준입니다. 6장

8.3.4 Stagnation

루프는 돌지만 같은 질문과 실패가 반복돼 실질적인 진전이 없는 상태입니다. 7장

8.3.5 Ralph

완료 기준을 충족하거나 중단 조건에 도달할 때까지 배경에서 수정과 검증을 반복하는 실행 방식입니다. 6장

8.4 기록과 ID

8.4.1 EventStore

모든 실행 이벤트가 기록되는 로컬 데이터베이스입니다. ~/.ouroboros/에 저장되며, ooo resume-session이 이를 직접 읽어 중단된 세션을 찾습니다. 7장

8.4.2 job_id · session_id · execution_id · lineage_id

실행이 돌려주는 네 가지 ID입니다. job_id는 배경 작업의 상태·결과 조회에, session_id는 status·evaluate와 세션 재개에, execution_id는 cancel에, lineage_id는 evolve 계보 확인과 ralph 입력에 사용합니다. 5장

8.4.3 auto_session_id

ooo auto가 중단될 때 남기는 자동 세션 ID입니다. ooo auto --resume에 넣어 같은 세션을 재개합니다. 7장
