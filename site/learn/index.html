한국어 입문 가이드

AI 코딩 작업을
시작 전에 정리하고
끝난 뒤 확인

Ouroboros는 만들 내용을 먼저 정리하고, AI가 그 기준에 따라 작업하게 하며, 결과가 기준을 충족했는지 확인하는 오픈소스 도구입니다.

입력과 결과.
사용자가 아이디어나 기존 저장소를 제공합니다. Ouroboros는 질문, 작업 기준 작성, 실행, 결과 확인을 관리합니다. 코드, 테스트 결과, Seed, 작업 기록이 남습니다.

1. Ouroboros가 하는 일

AI 코딩 도구에 요청을 바로 넘기지 않습니다. 먼저 빠진 결정을 질문하고, 답을 Seed라는 작업 기준으로 정리합니다. Claude Code나 Codex 같은 코딩 도구는 Seed를 기준으로 코드를 작성합니다. 작업이 끝나면 테스트와 완료 기준을 확인합니다.

1. 요청만들 기능이나 고칠 문제를 설명합니다.
2. 질문저장 방식, 사용자 범위, 완료 조건처럼 빠진 결정을 확인합니다.
3. 작업 기준답을 목표, 제약, 완료 기준이 있는 Seed로 저장합니다.
4. 실행연결한 AI 코딩 도구가 Seed를 기준으로 작업합니다.
5. 검증테스트 결과와 완료 기준 충족 여부를 확인합니다.
6. 수정찾은 문제를 다음 Seed에 반영합니다.

2. 사용하는 이유

2.1재작업 감소코딩 전에 빠진 결정을 확인합니다.
2.2완료 기준 고정무엇이 끝난 상태인지 문장과 테스트로 남깁니다.
2.3작업 기록 유지무엇을 실행했고 왜 실패했는지 다시 확인할 수 있습니다.
2.4AI 도구 교체같은 Seed를 여러 코딩 도구에서 사용할 수 있습니다.
2.5다음 작업에 반영검증에서 찾은 문제를 다음 명세에 포함합니다.

3. 전체 구조

3.1 명령ooo 명령으로 질문, 실행, 평가를 시작합니다.
3.2 Core질문, Seed, 정책, 실행 순서를 관리합니다.
3.3 RuntimeClaude Code, Codex 등이 실제 파일과 코드를 다룹니다.
3.4 Seed목표, 제약, 완료 기준을 담은 실행 기준입니다.
3.5 기록EventStore와 Ledger가 작업 과정과 결과를 저장합니다.
3.6 평가테스트와 기준 대조로 결과를 확인합니다.

4. 한 번 실행한 뒤 남는 결과

코드만 남지 않습니다. 실행에 사용한 Seed, 실행 상태, 테스트 결과, 평가 결과, 다음 수정에 사용할 정보가 함께 남습니다. 중단된 작업을 찾거나 같은 조건으로 다시 실행할 때 이 기록을 사용합니다.

적합한 작업.
요구사항이 여러 개이거나, 완료 여부를 증명해야 하거나, 여러 차례 수정할 작업에 적합합니다. 설명이 필요 없는 한 줄 수정에는 일반 AI 코딩 도구만 사용하는 편이 빠를 수 있습니다.

5. 가이드 구성

1장에서 제품의 역할과 구조를 확인하고 2장에서 설치합니다. 3장부터 작은 할 일 앱을 예제로 사용해 Interview, Seed, 실행, 평가, 수정을 순서대로 진행합니다.
