01 · 제품 개요

1. Ouroboros 이해

Ouroboros는 AI 코딩 작업의 요구사항, 실행, 검증, 기록을 한 흐름으로 관리하는 오픈소스 도구입니다.

Ouroboros 0.50.4upstream 456a134Ubuntu 24.04 · Claude Code 2.1.2102026-07-16 확인

1.1 무엇을 하는 도구인가

사용자는 만들 기능이나 고칠 문제를 설명합니다. Ouroboros는 바로 코드를 만들지 않고, 결과를 바꾸는 결정이 빠졌는지 질문합니다. 사용자의 답은 Seed라는 명세에 저장됩니다. 연결된 AI 코딩 도구는 Seed를 기준으로 작업하고, 작업이 끝나면 테스트와 완료 기준을 확인합니다.

구분내용
사용자가 제공하는 것아이디어, 기존 저장소, 제약사항, 질문에 대한 답
Ouroboros가 하는 것질문, Seed 작성, 실행 관리, 상태 기록, 결과 평가
AI 코딩 도구가 하는 것파일 작성, 코드 수정, 명령 실행, 테스트 수행
작업 후 남는 것코드, Seed, 테스트 결과, 평가 결과, 실행 기록

1.2 왜 사용하는가

“할 일 앱을 만들어 달라”는 요청에는 저장 방식, 사용자 수, 완료 항목 처리, 화면 범위가 없습니다. AI가 이 값을 임의로 정하면 코드를 만든 뒤 다시 고쳐야 합니다.

Ouroboros는 코딩 전에 필요한 결정을 질문합니다. 긴 기획서를 먼저 작성하라는 뜻은 아닙니다. 결과에 영향을 주는 선택만 확인하고, 확인한 내용을 실행 기준으로 남깁니다.

문제Ouroboros가 처리하는 방법효과
요청에 빠진 결정이 있음Interview에서 필요한 내용을 질문코딩 후 재작업 감소
AI가 완료했다고만 보고함완료 기준과 테스트 결과를 대조완료 여부를 직접 확인 가능
중간에 작업이 중단됨세션과 실행 상태를 기록중단 지점 확인과 재개 가능
다른 AI 도구로 바꾸기 어려움Seed와 Core를 runtime에서 분리같은 기준을 여러 도구에서 사용
같은 실패를 반복함평가 결과를 다음 Seed에 반영다음 실행의 요구사항 개선

1.3 전체 구조

Ouroboros Core가 작업 순서와 기록을 관리하고, Claude Code나 Codex 같은 runtime이 실제 코드를 작성합니다. Seed는 두 부분을 연결하는 작업 기준입니다.

1.3.1 ooo 명령Interview, 실행, 평가 같은 기능을 시작합니다.
1.3.2 Core질문, Seed, 정책, 실행 순서, 평가를 관리합니다.
1.3.3 RuntimeClaude Code, Codex 등이 파일과 코드를 다룹니다.
1.3.4 Seed목표, 제약, 완료 기준을 담습니다.
1.3.5 EventStore세션, 실행 상태, 평가 결과를 저장합니다.
1.3.6 Evaluator테스트 결과와 완료 기준 충족 여부를 확인합니다.
역할 구분.
Ouroboros는 코드를 직접 작성하는 모델이 아닙니다. 작업 기준과 순서를 관리하는 계층입니다. 실제 코드 작성은 연결한 runtime이 담당합니다.

1.4 한 번 작업하는 순서

  1. 아이디어 입력. 만들 기능이나 고칠 문제를 설명합니다.
  2. Interview. 결과에 영향을 주는 빠진 결정을 질문합니다.
  3. Seed 확인. 목표, 제약, 완료 기준이 사용자 의도와 맞는지 확인합니다.
  4. 실행. runtime이 Seed를 기준으로 코드를 작성하고 테스트합니다.
  5. 평가. 테스트 결과와 완료 기준을 확인합니다.
  6. 수정. 찾은 문제를 다음 Seed에 반영합니다.

작은 할 일 앱을 예로 들면, 사용자가 기능을 설명하고 저장 방식과 완료 처리에 답합니다. Ouroboros가 이 답을 Seed로 저장합니다. runtime이 앱을 만들고, 새로고침 후 데이터 유지와 완료 표시가 실제로 작동하는지 확인합니다.

1.5 주요 이점

  • 요구사항 유지. 대화가 길어져도 확정한 목표와 제약을 Seed에서 다시 확인할 수 있습니다.
  • 완료 기준 확인. “완료”라는 보고 대신 테스트와 기준 충족 여부를 확인합니다.
  • 작업 재개. 세션과 실행 ID를 사용해 중단된 작업을 찾습니다.
  • 실행 재현. 같은 Seed와 정책으로 다시 실행할 수 있습니다.
  • 도구 선택. 작업 기준을 유지한 채 Claude Code, Codex 등 runtime을 바꿀 수 있습니다.

1.6 이 가이드의 예제

오늘 할 일을 추가하고 완료 표시하는 웹앱을 사용합니다. 기능은 작지만 저장, 완료 처리, 사용자 범위, 테스트 기준을 정해야 하므로 Ouroboros의 흐름을 확인하기에 적합합니다.

2. Ouroboros 설치

이 가이드는 Claude Code 플러그인을 기본 경로로 사용합니다. Claude Code 로그인, 플러그인 설치, Core 연결, 설치 확인 순서로 진행합니다.

2.1 Claude Code 실행과 로그인

Claude Code 설치를 마친 뒤 터미널에서 대화창을 엽니다.

claude

Claude Code 대화창에서 로그인합니다.

/login

로그인 후 터미널에서 버전을 확인합니다.

claude --version

2.2 Ouroboros 플러그인 설치

다음 두 명령은 일반 터미널에서 실행합니다.

claude plugin marketplace add Q00/ouroboros
claude plugin install ouroboros@ouroboros

2.3 작업 폴더에서 Core 연결

작업할 프로젝트 폴더에서 Claude Code를 열고, Claude Code 대화창에 다음 명령을 입력합니다.

ooo setup

setup은 Ouroboros Core의 MCP 서버를 등록합니다. 사용자가 선택하면 현재 프로젝트의 CLAUDE.md에 명령 요약도 추가합니다.

로컬 저장 위치.
~/.ouroboros/에 설정, Seed, EventStore, 로그가 저장됩니다. 프로젝트에는 선택에 따라 CLAUDE.md 요약과 .ouroboros/ 정보가 생길 수 있습니다.

2.4 설치 확인

Claude Code 대화창에서 도움말을 실행합니다.

ooo help

Interview, Seed, run, evaluate 명령이 표시되면 플러그인이 로드된 것입니다. setup 결과에서 MCP 서버 등록을 확인한 뒤 Claude Code를 다시 시작합니다.

2.5 업데이트

ooo update

현재 버전과 최신 버전을 비교하고 설치 방식에 맞춰 패키지와 runtime 연결을 갱신합니다. 업데이트 후 Claude Code 세션을 다시 엽니다.

다른 runtime 설치

독립 터미널 경로는 Python 3.12 이상이 필요합니다. 설치 후 사용할 runtime을 지정합니다.

pip install ouroboros-ai
ouroboros setup --runtime codex

본문의 명령은 Claude Code의 ooo 형식을 기준으로 설명합니다.
