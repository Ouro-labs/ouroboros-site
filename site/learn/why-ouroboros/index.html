2. 왜 사용하는가

AI 코딩 도구는 빠르지만, 요구사항 확정과 완료 검증은 사용자에게 남습니다. Ouroboros는 그 두 가지를 절차로 만듭니다.

Ouroboros 0.50.4upstream 456a1342026-07-16 문서·코드 확인

2.1 AI 코딩 도구만 사용할 때 생기는 문제

"할 일 앱을 만들어 달라"는 요청에는 저장 방식, 사용자 수, 완료 처리 방법이 없습니다. AI가 이 값을 임의로 정하면 코드가 나온 뒤에 고쳐야 합니다.

문제결과
빠진 결정을 AI가 임의로 채움코드 완성 후 재작업
"완료했습니다" 보고만 남음실제 완료 여부를 확인할 방법이 없음
대화가 길어지면 초기 요구가 흐려짐처음 합의와 다른 결과
세션이 끊기면 맥락이 사라짐중단 지점을 찾지 못하고 처음부터 다시 설명
같은 실패를 반복해도 기록이 없음수정 방향이 누적되지 않음

2.2 Ouroboros를 사용하면 달라지는 점

변화방법
코딩 전에 결정을 확정Interview가 빠진 결정을 질문합니다(5장)
완료 기준을 문서로 고정완료 기준(Acceptance Criteria)이 Seed에 저장됩니다(6장)
완료 여부를 검증테스트와 기준 대조로 3단계 평가를 합니다(8장)
중단 후 재개실행 이벤트가 로컬 DB에 남아 세션을 복구합니다(11장)
실패를 다음 실행에 반영평가 결과로 Seed를 고쳐 다시 실행합니다(9장)

2.3 적합한 작업

  • 요구사항이 여러 개라 정리가 필요한 기능 개발
  • 완료 여부를 테스트로 증명해야 하는 작업
  • 여러 차례 수정을 거칠 것이 예상되는 작업
  • 중간에 끊겨도 이어서 진행해야 하는 긴 작업

2.4 사용하지 않아도 되는 작업

오타 수정이나 한 줄 변경처럼 설명이 필요 없는 작업은 AI 코딩 도구만 사용하는 편이 빠릅니다. 질문과 평가 절차가 작업 자체보다 커지기 때문입니다.
