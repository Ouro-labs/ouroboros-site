10. 기존 프로젝트에 적용

9장까지는 새 예제 프로젝트였습니다. 이 장은 이미 코드가 있는 저장소와 팀 작업에 같은 절차를 적용하는 방법입니다.

Ouroboros 0.50.4upstream 456a1342026-07-16 문서·코드 확인

10.1 무엇이 다른가

기존 저장소에서는 이미 정해진 기술과 제약이 있습니다. 이를 등록해 두면 Interview가 새 프로젝트처럼 추측하지 않고, 현재 코드와 제약을 바탕으로 질문합니다.

10.2 brownfield — 기존 저장소 등록

Claude Code 대화창에서 실행합니다.

ooo brownfield

정상 결과: git 저장소와 worktree 목록이 번호와 함께 표시되고, Interview의 기본 컨텍스트로 쓸 저장소를 선택합니다. 실제 작업할 저장소만 기본값으로 선택합니다.

공식 문서 기준의 하위 명령입니다.

명령역할
ooo brownfield scan저장소 검색만 수행
ooo brownfield defaults현재 기본 저장소 확인
ooo brownfield set 6,18번호로 기본 저장소 지정

10.3 pm — 제품 요구 정리

기능 하나가 아니라 제품 단위의 요구를 정리할 때 사용합니다. Claude Code 대화창에서 실행합니다.

ooo pm

정상 결과: 제품 관점의 질문이 진행되고 PRD(제품 요구사항 문서)가 만들어집니다.

PRD와 Seed는 다릅니다. PRD는 제품이 무엇이어야 하는지 적은 문서이고, Seed는 한 번의 실행이 따를 작업 기준입니다. PRD는 그대로 실행되지 않으며, 그 내용을 ooo interview에 넣어 실행 단위의 Seed를 다시 만듭니다.

10.4 publish — 팀 작업 발행

Seed를 팀이 나눠 작업할 GitHub Issues로 바꿉니다. gh CLI 설치와 로그인이 필요합니다. Claude Code 대화창에서 실행합니다.

ooo publish <seed_path>

정상 결과: 생성될 이슈 구조가 먼저 표시되고, 확인 후 대상 저장소에 이슈가 만들어집니다.

10.5 완료 기준

  • brownfield 기본 저장소가 실제 작업할 저장소로 설정되어 있다.
  • Interview 질문이 기존 코드의 제약을 반영한다.
  • 팀 작업이 필요한 경우, Seed 기반 이슈가 대상 저장소에 생성됐다.
